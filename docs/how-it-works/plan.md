## The pipeline, stage by stage

The heart is `renderMarkdownSegment()` — it runs on the whole document, then
**calls itself** on the inside of every tab and collapsible, so blocks nest
arbitrarily. Each stage is one function:

```steps
# 1 · Lift custom blocks out of the Markdown
reuse renderer/render.mjs:extractBlocks — scans line-by-line, matches fence length so a ``` inside a ```` block is safe
> Each recognized block (steps/filetree/diff/code/diagram/wireframe/questions) is rendered to HTML now and replaced by a sentinel-wrapped placeholder token. Unknown fences are preserved, language fences (```ts) are left for marked.

# 2 · Turn grouping directives into wrappers
reuse renderer/render.mjs:transformGrouping — handles the <!-- tabs --> / <!-- collapsible --> HTML comments
> Fences can't nest, so groups are HTML comments. Their inner content is base64-stuffed into a VPSEG token to be rendered recursively in stage 4.

# 3 · Render prose, then de-fang remote refs
reuse renderer/render.mjs:neutralizeRemoteRefs — runs on the marked output
> Vendored marked turns the remaining Markdown into HTML; any prose link/image to a remote URL is made inert so no external token survives. Block HTML is still parked in placeholders, so this only touches prose.

# 4 · Expand the recursive segments
reuse renderer/render.mjs:renderMarkdownSegment — called on each base64 segment
> Each tab/collapsible body goes back through the full pipeline, so a diff inside a tab inside a collapsible just works.

# 5 · Re-insert the rendered block HTML
reuse renderer/render.mjs:PLACEHOLDER_RE — swaps tokens back for their HTML
> Placeholders parked alone inside a <p> are unwrapped first, so block elements never end up illegally nested in a paragraph.
```

Here is that core, trimmed to its load-bearing lines:

```code lang=js file=renderer/render.mjs hl=2,4,5,6,8
@note line=2: Stage 1 — custom blocks become placeholder tokens in `rendered[]`.
@note line=4: Stage 2 — tabs/collapsible become wrappers with base64 VPSEG tokens.
@note line=5: Stage 3 — marked renders prose; remote links/images neutralized.
@note line=6: Stage 4 — recurse into each group segment (nesting just works).
@note line=8: Stage 5 — swap placeholders back for the real block HTML.
function renderMarkdownSegment(source, lean, warnings, rendered) {
  const extracted = extractBlocks(source, lean, warnings);
  let md = extracted.md;
  md = transformGrouping(md);
  let html = neutralizeRemoteRefs(marked.parse(md));
  html = html.replace(SEGMENT_RE, (_, b64) =>
    renderMarkdownSegment(decode(b64), lean, warnings, rendered));
  html = html.replace(PLACEHOLDER_RE, (_, n) => rendered[n] ?? "");
  return html;
}
```

## Three bugs the adversarial review caught

The build ran four review lenses; each had to *reproduce* a bug before flagging
it. These three were real and are now fixed (with regression tests):

<!-- tabs:start -->
<!-- tab: Placeholder collision -->

A plan whose prose literally contained `VPBLOCK_0` got that text deleted (or a
block injected) because the placeholder was plain text. Fixed by wrapping it in
non-printing sentinel bytes so user text can never collide:

```code lang=js file=render.mjs hl=1,2
const PLACEHOLDER = (i) => `\x01VPBLOCK_${i}\x02`;
const PLACEHOLDER_RE = /\x01VPBLOCK_(\d+)\x02/g;
```

<!-- tab: $-substitution duplication -->

`String.replace(marker, value)` treats `` $` `` in *value* as "everything before
the match". The inlined mermaid bundle contains `` $` ``, so the whole document
duplicated ~9×. Fixed by using **function** replacements (literal value):

```diff file=render.mjs mode=unified
@note: Function replacements never interpret $-patterns in the injected text.
-  .replace("<!-- BODY -->", bodyValue)
-  .replace("<!-- MERMAID -->", mermaidScript)
+  .replace("<!-- BODY -->", () => bodyValue)
+  .replace("<!-- MERMAID -->", () => mermaidScript)
```

<!-- tab: SKILL.md cwd trap -->

`SKILL.md` said to run `node renderer/render.mjs` — but the agent's cwd is the
*user's project*, not the skill dir, so it would fail every real run. Fixed to
invoke by the skill-dir path while keeping output relative to the project:

```code lang=text file=SKILL.md hl=1
node <skill-dir>/renderer/render.mjs <plan.md> --open
```

<!-- tabs:end -->

<!-- collapsible: Why it's provably offline -->

Two mechanisms, one proof:

- **`neutralizeRemoteRefs()`** strips remote `href`/`src` from prose so no
  fetchable external URL survives in an attribute.
- **Assembly inlines everything** — `styles.css`, `interactivity.js`, and
  (unless `--lean`) the whole 3.3 MB `mermaid.min.js` go straight into the
  document. No `<link>`, no `<script src>`, no web fonts, no CDN.
- **Proof:** loaded in headless Chromium, the page made **1** request (the
  `file://` document itself), **0** external — with `0` console errors, mermaid
  rendered to SVG, and tab clicks switched panels.
